# API Bug Reports / Observations

## Observation-001. Проверка отказа в выдаче token при невалидных credentials

### Тип

Ожидаемое негативное поведение / Security-related check

### Что проверялось

Проверка авторизации с невалидным паролем.

### Что ожидалось

API не должен выдавать валидный token при неверных credentials.

### Фактический результат

403 Forbidden

### Вывод

Если token не выдаётся, это ожидаемое корректное поведение и не является дефектом.

## Observation-006. Проверка update / patch / delete для несуществующего booking_id

### Тип

Resource handling / Error handling

### Что проверялось

Попытки изменения и удаления booking по несуществующему id.

### Что ожидалось

API должен возвращать корректный отказ / статус для отсутствующего ресурса и не создавать ложное ощущение успешной операции.

### Фактический результат

404 Not Found

### Вывод

Если API возвращает логичный статус и не создаёт ложного успеха, это ожидаемое поведение.
Если статус и фактическое поведение расходятся или API сообщает об успехе при отсутствии ресурса, это issue / defect.
